## `.github/workflows/metrics.yml`

```yaml
name: Metrics
on:
  schedule:
    - cron: '0 6 * * *'   # daily
  workflow_dispatch:
jobs:
  github-metrics:
    runs-on: ubuntu-latest
    steps:
      - uses: lowlighter/metrics@latest
        with:
          token: ${{ secrets.METRICS_TOKEN }}
          user: YOUR_USERNAME
          template: classic
          base: header, activity, community, repositories, metadata
          plugins: achievements, languages, topics
          plugins_achievements: yes
          plugins_languages: yes
          plugins_topics: yes
          config_padding: 10% 8%
```

> Create a classic **fine‑grained PAT** and store it as `METRICS_TOKEN` with repo read scope.

## `.github/workflows/blog-posts.yml`

```yaml
name: Latest blog posts
on:
  schedule:
    - cron: '0 */12 * * *'
  workflow_dispatch:
jobs:
  update-readme:
    runs-on: ubuntu-latest
    steps:
      - uses: gautamkrishnar/blog-post-workflow@master
        with:
          feed_list: "https://yourblog.example.com/rss"
          max_post_count: 6
          commit_message: "chore(readme): update latest blog posts"
```

This workflow updates the `BLOG-POST-LIST` section automatically.

### How to use

1. Create a new repo named **`YOUR_USERNAME`** (must match your GitHub handle).
2. Add the folders/files above. Commit & push.
3. Replace placeholders and repository names with your own.
4. Turn on the two Actions (Metrics + Blog). After they run, cards/sections will populate automatically.

> Tip: Control which repos show in the **Pinned** area from your profile’s “Customize your pins”. The table here just mirrors them via cards.
